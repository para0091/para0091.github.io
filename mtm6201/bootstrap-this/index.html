Some of them are: .text-primary , .text-secondary , .text-success , .text-danger , .text-warning , .text-info , .text-light , .text-dark , .text-body , .text-muted , .text-white.

Display

We can apply display properties to elements by using .d-{value} classes.

.d-{value} is used for xs breakpoint while .d-{breakpoint}-{value} is used for other breakpoints.

The values are: none, inline, inline-block, block, grid, table, table-cell, table-row, flex and inline-flex.

Simply apply different display classes to toggle the display value at different breakpoints if needed.

Position

Position utility classes can be used to configure the positioning of an element. Eg. .position-static.

To arrange the elements, we use {property}-{position}, where property is [top, bottom] for y axis and [start, end] for x axis. and position is the value. We can use the value 0 for 0 edge position, 50 for 50% edge position and 100 for 100% edge position.

To center elements, use the class .translate-middle. Also use .translate-middle-x or .translate-middle-y as needed.

Spacing

We can adjust the margin, padding and gap properties by using their corresponding utility classes.

The format to follow to apply margin and padding is {property}{sides}-{size}, where property is m or p, which is margin or padding.
sides are: t=top, b=bottom, s=start, e=end, x=left+right, y=top+bottom and blank.
size is one of: 0,1,2,3,4,5 or auto.

Another spacing class we can use is gap, which lets us add a gap between individual grid items, but we have to be using display: grid; (d-grid).

Layout

Containers

Containers are is used to keep all the content inside it in a specific maximum width for each breakpoint. There are default, fluid and responsive containers.
Default [.container]: This container is resposive and it changes the max-width according to the breakpoints.
Fluid [.container-fluid]: This is a container that takes up the full width of the viewport.
Responsive: .container-sm, .container-md, .container-lg, .container-xl, .container-xxl

Components

Navigation Bar

Bootstrap has a navigation bar component which is very powerful and useful.

Some of the main ideas are:

Nav tags have to be applied a class of .navbar for default navbar and .navbar-expand + {-sm/-md/-lg/-xl/-xxl} for responsive navbars.
We can edit the navbar by using flex and spacing classes as well as editing the container.
We can even use a form in the navbar if needed. In addition to that, we can simply use a tags only instead of using lists. Adding a dropdown menu to the navbar is also quite easy.

We can use different background colors and theme the navbar as we like.
Use position utility classes to edit the positioning of the navbar. For eg, to stick the navbar to the top, we can use the class sticky-top.

Cards

Cards in Bootstrap are small containers of content. They can have images, text, lists, links and even a header and footer.

We can nest cards in rows and use the grid system to control multiple cards. In addition, we can style the cards as we like using Bootstrap utilities as well as Custom CSS.

We can even add navigation inside cards using Bootstrap's nav components.

Multiple cards can be grouped together with the help of Card layouts.

It is also easy to change up the color schemes and text alignment and we can even play around with the images.
